# Twinssistant

A personal AI assistant ("my twin") with voice conversation, multi-provider LLM routing (OpenAI / Anthropic / Gemini), web search with Tavily, Google Calendar management, weather-aware outfit suggestions, and a CV review/rewrite pipeline.

## Structure

- `my-twin/` — React + Vite frontend (MUI, Tailwind, Firebase Auth, Socket.IO client)
- `my-twin/Assistbackend/` — Flask backend (REST + Socket.IO), provider routing in `Routing.py`, intent caching with Pinecone, ElevenLabs TTS

## Setup

### Backend

```bash
cd my-twin/Assistbackend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

Create `my-twin/Assistbackend/.env` with:

```
OPENAI_API_KEY=...
CLAUDE_API_KEY=...
GEMINI_API_KEY=...
TAVILY_API_KEY=...
OPENWEATHER_API_KEY=...
PINECONE_API_KEY=...
ELEVENLABS_API_KEY=...
```

Also required in `my-twin/Assistbackend/` (never commit these):

- `tw.json` — Firebase service account key
- `credentials.json` — Google OAuth client (from Google Cloud Console)
- `token.json` — generated by running `python login_assist.py` once (Google Calendar OAuth)

Run:

```bash
python Assist.py   # serves on http://127.0.0.1:5000
```

### Frontend

```bash
cd my-twin
npm install
npm run dev        # serves on http://localhost:5173
```

Optional `my-twin/.env`:

```
VITE_API_BASE_URL=http://127.0.0.1:5000
VITE_AI_SPEECH_WS_URL=http://localhost:5000
```

## Notes

- Backend API routes require a Firebase ID token (`Authorization: Bearer <token>`); the frontend attaches it automatically once logged in.
- `Cv_docs/`, `chat/`, and `refined/` are runtime data directories (uploads, chat history, generated files) and are gitignored.
- See `CODE_REVIEW.md` for the current improvement roadmap.
